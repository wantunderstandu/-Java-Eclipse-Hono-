需求分析：
1. 设备注册
2. 设备凭证配置
3. 设备列表查询
4. 命令下发

固定租户由后端配置，不让前端传 `tenantId`。

```yaml
hono:
  tenant-id: demo-tenant
```

**接口设计**
**1. 设备注册**

```http
POST /api/devices
Content-Type: application/json
```

Request Body：

```json
{
  "deviceId": "device-001",
  "name": "温湿度传感器-001",
  "description": "实验室一号传感器",
  "enabled": true
}
```

说明：

`deviceId` 是设备唯一标识。  
`name` 和 `description` 可以存入 Hono 设备扩展字段。  
`enabled` 表示设备是否启用。

返回示例：

```json
{
  "success": true,
  "message": "设备注册成功",
  "data": {
    "deviceId": "device-001",
    "tenantId": "demo-tenant",
    "enabled": true
  }
}
```

**2. 设备凭证配置**

```http
POST /api/devices/{deviceId}/credentials
Content-Type: application/json
```

示例：

```http
POST /api/devices/device-001/credentials
```

Request Body：

```json
{
  "authId": "device-001",
  "password": "123456"
}
```

说明：

`authId` 是设备认证 ID，MVP 中默认可以和 `deviceId` 保持一致。  
`password` 是设备连接 Hono 时使用的密码。  
后端收到明文密码后，应调用 Hono Credentials API 进行凭证配置，不建议自己长期保存明文密码。

返回示例：

```json
{
  "success": true,
  "message": "设备凭证配置成功",
  "data": {
    "deviceId": "device-001",
    "authId": "device-001"
  }
}
```

**3. 设备列表查询**

```http
GET /api/devices
```

MVP 版本可以不设计 JSON Request Body，因为这是查询接口。

可选查询参数：

```http
GET /api/devices?page=0&size=20
```

返回示例：

```json
{
  "success": true,
  "message": "查询成功",
  "data": {
    "tenantId": "demo-tenant",
    "total": 2,
    "devices": [
      {
        "deviceId": "device-001",
        "name": "温湿度传感器-001",
        "description": "实验室一号传感器",
        "enabled": true
      },
      {
        "deviceId": "device-002",
        "name": "智能开关-002",
        "description": "实验室照明控制设备",
        "enabled": true
      }
    ]
  }
}
```

**4. 命令下发**

```http
POST /api/devices/{deviceId}/commands
Content-Type: application/json
```

示例：

```http
POST /api/devices/device-001/commands
```

Request Body：

```json
{
  "commandName": "setTemperature",
  "contentType": "application/json",
  "payload": {
    "targetTemperature": 26
  },
  "timeoutSeconds": 10
}
```

也可以发送简单字符串命令：

```json
{
  "commandName": "switch",
  "contentType": "text/plain",
  "payload": "ON",
  "timeoutSeconds": 10
}
```

说明：

`commandName` 是命令名称。  
`contentType` 表示命令内容格式。  
`payload` 是实际命令内容，可以是 JSON 对象或字符串。  
`timeoutSeconds` 是等待命令下发结果的超时时间。MVP 中可以只返回“命令发送请求已提交”，不强制实现设备响应处理。

返回示例：

```json
{
  "success": true,
  "message": "命令下发请求已提交",
  "data": {
    "deviceId": "device-001",
    "commandName": "setTemperature",
    "status": "SENT"
  }
}
```

**最终 MVP 接口汇总**

```text
POST /api/devices
设备注册

POST /api/devices/{deviceId}/credentials
设备凭证配置

GET /api/devices
设备列表查询

POST /api/devices/{deviceId}/commands
命令下发
```
